P & NP

  • P: class of problems solvable in polynomial time. O(nk)O(n^k)O(nk) for some constant kkk.

    Shortest paths in a graph.

  • NP: class of problems verifiable in polynomial time.

    Verifying a cycle is hamiltonian.

  • NP-complete: problem is in NP and is as hard as any problem in NP.

    如果NPC问题可在多项式时间内被解决，那么所有的NP问题都可以在多项式时间内被解决

Interval Scheduling / 区间调度问题

Requests 1,2,...,n1,2, . . . , n1,2,...,n, single resource

s(i)s(i)s(i) start time, f(i)f(i)f(i) finish time, s(i)<f(i)s(i) < f(i)s(i)<f(i) (start time must be less than finish time for a request)

Two requests iii and jjj are compatible if they don’t overlap, i.e., f(i)≤s(j)f(i) \le s(j)f(i)≤s(j) or f(j)≤s(i)f(j) \le s(i)f(j)≤s(i).

Goal: Select a compatible subset of requests of maximum size.

Greedy Algorithm

贪心算法是一种在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是最好或最优的算法。

  1. 使用某种方法选择一个request iii.
  2. 拒绝request中冲突的部分
  3. 重复这一步骤直到所有的request被处理

Rules

Claim 1. Greedy algorithm outputs a list of intervals

<s(i1),f(i1)>,<s(i2),f(i2)>,...,<s(ik),f(ik)>< s(i_1), f(i_1)>, < s(i_2), f(i_2)>, . . . , < s(i_k), f(i_k) > <s(i1​),f(i1​)>,<s(i2​),f(i2​)>,...,<s(ik​),f(ik​)>

such that

s(i1)<f(i1)≤s(i2)<f(i2)≤…≤s(ik)<f(ik)s\left(i_{1}\right)<f\left(i_{1}\right) \leq s\left(i_{2}\right)<f\left(i_{2}\right) \leq \ldots \leq s\left(i_{k}\right)<f\left(i_{k}\right) s(i1​)<f(i1​)≤s(i2​)<f(i2​)≤…≤s(ik​)<f(ik​)

Claim 2. Given list of intervals LLL, greedy algorithm with earliest finish time produces k∗k^*k∗ intervals, where k∗k^*k∗ is optimal.

证明可使用数学归纳法，从k∗=1k^*=1k∗=1的base case归纳，可以得到k∗k^*k∗一直是最优的序列，详细过程略。

Weighted Interval Scheduling

Each request iii has weight w(i)w(i)w(i). Schedule subset of requests that are non-overlapping with maximum weight.

我们不难发现，在此时贪心算法是失效的，因此我们选择动态规划解决。

Dynamic Programming

动态规划（DP）是一种通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。

我们这样定义Weighted Interval Scheduling的子问题：

Rx={j∈R∣s(j)≥x}R^{x}=\{j \in R \mid s(j) \geq x\} Rx={j∈R∣s(j)≥x}

其中，RRR是所有request的集合，如果我们令x=f(i)x=f(i)x=f(i)，那么RxR^{x}Rx为集合RRR中起始时间晚于request iii的request，这样我们需要解决的总子问题数n=∣R∣n=|R|n=∣R∣且只需要计算并记录一次。

当我们选择request iii作为序列的起始时，那么剩余的request集合为Rf(i)R^{f(i)}Rf(i)，不难注意到有些request与request iii是协调的，但它仍然不会出现在Rf(i)R^{f(i)}Rf(i)中，即我们在按序遍历所有的子问题：

opt⁡(R)=max⁡1≤i≤n(w(i)+opt⁡(Rf(i)))\operatorname{opt}(R)=\max _{1 \leq i \leq n}\left(w(i)+\operatorname{opt}\left(R^{f(i)}\right)\right) opt(R)=1≤i≤nmax​(w(i)+opt(Rf(i)))

此时，时间复杂度为O(n2)O(n^2)O(n2)，这是由于解决每个子问题的时间复杂度为O(n)O(n)O(n).
